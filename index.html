📖 Roadmap Guide

Eftsure + Sis ID · 2026 · Powered by ProductBoard

Views

  • By Region — columns show ANZ, US & Global, Europe. Features appear once with secondary region badges.
  • By Quarter — columns show quarters (Q1–Q4). Cards display region badge instead of timeline.
  • By Status — columns show Candidate, Planned, In Progress, Released. Cards display region and timeline.
  • Ranked List — flat table of all features. Score each one using the EFTsure Prioritisation Matrix (Strategic Fit, Revenue, Cost, Risk, Confidence, Effort). ROI Score auto-calculates and re-ranks on every change. Scores are saved locally.

Filters

  • Team — toggle EFTsure (orange) or Sis ID (purple) on/off.
  • Status — filter by Candidate, Planned, In Progress, Released.
  • Region — show only features in selected regions. Works in both tabs.
  • Quarter — show only features with a timeframe in selected quarters. Features without a timeframe always show.
  • Click any chip to deactivate; click again to restore.

Cards

  • Click a card to open the feature directly in ProductBoard.
  • Drag up/down to reorder, or across columns to reassign region/quarter (session only).
  • × button (hover to reveal) hides a card. Refresh restores all hidden cards.
  • Status dot — blue = Candidate · amber = Planned · orange = In Progress · green = Released.
  • Health badge — ● On track · ⚠ At risk · ✖ Off track (In Progress features only).
  • Border colour — orange = EFTsure · purple = Sis ID.

Data & Sync

  • ↻ Refresh — reloads the latest saved data from the database. Fast, no ProductBoard API call.
  • ⇄ Sync — fetches fresh data from ProductBoard, saves to database, updates the view. Run after updating features in PB.
  • Eftsure: Live data — EFTsure features are live-synced from ProductBoard.
  • Sis ID: DB — Sis ID features are manually maintained in the database.
  • Unmapped features — collapsible section at the bottom lists features missing an initiative or timeframe in ProductBoard.
